# Futuristic Disk Scheduling Simulator

A high-performance, interactive, sci-fi themed disk scheduling algorithm simulator built with **WebAssembly (C → WASM)**, Vanilla JavaScript, and Vite.

##  Features

- **8 Algorithms**: FCFS, SSTF, SCAN, LOOK, C-SCAN, C-LOOK, FSCAN, and N-Step SCAN.
- **WASM-Powered**: Core scheduling algorithms written in C and compiled to WebAssembly for near-native performance.
- **Direction Control**: Configurable initial head direction (LEFT / RIGHT) for directional algorithms (SCAN, LOOK, C-SCAN, C-LOOK, FSCAN, N-Step SCAN).
- **Step Size Parameter**: Adjustable batch size (N) for the N-Step SCAN algorithm.
- **Interactive Timeline**: Scrubbable timeline to replay execution step-by-step.
- **Visualizations**: 
    - **Linear Graph**: Glowing path animations showing head movement.
    - **Radar View**: Circular disk representation.
- **Educational Mode**: Learn about each algorithm with descriptions, pros, and cons.
- **Cyberpunk UI**: Glassmorphism, neon glows, and smooth motion with Orbitron & Rajdhani fonts.

##  Setup & Run

### Prerequisites

- **Node.js** (v16+)
- **Emscripten SDK** (for rebuilding WASM — only needed if modifying C algorithms)

### Quick Start

1. **Install Dependencies**
   ```bash
   npm install
   ```

2. **Start Dev Server**
   ```bash
   npm run dev
   ```

3. **Open in Browser**  
   Click the link shown in the terminal (usually `http://localhost:5173`).

### Rebuilding WASM (after modifying C code)

```bash
./build.sh
```

This compiles all C algorithm files into a single `src/wasm/scheduler.mjs` module using Emscripten.

##  Algorithms Implemented

| Algorithm | Description | Direction | Extra Params |
|-----------|-------------|:---------:|:------------:|
| **FCFS** | First-Come, First-Served — serves requests in arrival order. | — | — |
| **SSTF** | Shortest Seek Time First — picks the closest request. | — | — |
| **SCAN** | Elevator Algorithm — sweeps end-to-end, servicing requests. | ✅ | — |
| **LOOK** | Adaptive SCAN — sweeps only as far as the last request. | ✅ | — |
| **C-SCAN** | Circular SCAN — one-way sweep, jumps back to start. | ✅ | — |
| **C-LOOK** | Adaptive Circular SCAN — one-way sweep to last request. | ✅ | — |
| **FSCAN** | Freeze-SCAN — freezes current queue, services with SCAN. | ✅ | — |
| **N-Step SCAN** | Batch SCAN — divides queue into sub-queues of size N, each processed with SCAN. | ✅ | Step Size (N) |

```
Disk-Scheduling-Algo/
├── index.html                  # Main HTML entry point
├── build.sh                    # Emscripten WASM build script
├── package.json                # Node.js project config
├── src/
│   ├── main.js                 # Application entry — UI wiring & event handling
│   ├── style.css               # Cyberpunk-themed CSS styles
│   ├── algorithms/             # C algorithm source files
│   │   ├── fcfs.c              # First-Come, First-Served

│   │   ├── scan.c              # SCAN (Elevator)

│   │   ├── cscan.c             # Circular SCAN
│   │   ├── clook.c             # Circular LOOK
│   │   ├── fscan.c             # Freeze-SCAN
│   │   ├── nstep_scan.c        # N-Step SCAN
│   │   ├── wasm_bridge.c       # WASM bridge — wraps C functions for JS
│   │   └── wasm_loader.js      # JS loader — provides high-level API to Engine
│   ├── scheduler/
│   │   └── scheduler.h         # Shared C header with function declarations
│   ├── simulation/
│   │   └── Engine.js           # Simulation engine — calculation, playback, seeking
│   ├── ui/
│   │   └── Renderer.js         # Canvas renderer — timeline & radar visualizations
│   └── wasm/
│       └── scheduler.mjs       # Compiled WASM module (generated by build.sh)
└── dist/                       # Production build output (generated by vite build)
```

##  Tech Stack

- **Vite** — Fast build tool and dev server.
- **Vanilla JS** — No frameworks, pure performance.
- **C + Emscripten** — Algorithms compiled to WebAssembly for speed.
- **HTML5 Canvas** — Smooth 60fps animations.
- **CSS3** — Custom properties, Flexbox/Grid, Backdrop Filter, glassmorphism effects.
- **Google Fonts** — Orbitron (display) & Rajdhani (UI text).
